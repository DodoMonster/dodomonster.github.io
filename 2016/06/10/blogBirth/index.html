Blog破蛋记

Veröffentlicht am 2016-06-10   |   in settings

前言

经过1天的折腾终于把此博客搭建好并上线了，真心想说句不容易，躺了好多坑呀。。。因此也作为本博客的第一篇正式的博客记录下来。

前提物质条件

所谓“经济基础决定上层建筑”（好吧。。。纯属瞎扯）!!!本博客是通过Hexo和github pages搭建的。在使用Hexo搭建博客前首先你要在自己电脑上安装配置好以下三样东西。

  • NodeJS环境：安装配置可参考教程 windows下配置安装NodeJS

  • git：安装配置可参考教程 windows下配置安装git

  • github账号：请自行打开 github官网 进行注册登录

安装Hexo

当你的电脑上node环境配置安装好，git和github配置好后，方可正常又顺利地按照以下步骤对hexo进行配置安装。

  1. 打开命令行cmd，安装Hexo： npm install -g hexo
  1. 查询Hexo是否安装成功及版本：hexo -v

配置Hexo

  1. 创建本地文件夹：HexoBlog(名字随你起的。。。)，此处文件夹路径为：F:\HexoBlog。命令行下进入该文件夹后输入hexo init

  1. 安装依赖和插件：npm install

  2. 生成静态化页面：hexo generate（或者hexo g）

4.启动Hexo服务：hexo server（或者hexo s）

5.在浏览器输入：http://localhost:4000

至此简单的本地Hexo安装配置就成功了。

创建github仓库

  1. 打开https://github.com 登录github，新建仓库:yourusername.github.io(这是固定的仓库名，把yourusername换为你的github昵称，如我的github昵称为dodomonster，所以我的仓库名为dodomonster.github.io)

  1. 返回hexoBlog文件夹，查看文件结构内容如下：

  • scaffolds 脚手架，也就是一个工具模板
  • scripts 写文件的js，扩展hexo的功能
  • source 存放博客正文内容
  • source/_drafts 草稿箱
  • source/_posts 文件箱
  • themes 存放皮肤主题的目录
  • themes/landscape 默认的皮肤
  • _config.yml 全局的配置文件
  • db.json 静态常量

在这里，我们每次用到的就是_posts目录里的文件，而_config.yml文件和themes目录是第一次配置好就行了。

_posts目录：Hexo是一个静态博客框架，因此没有数据库。文章内容都是以文本文件方式进行存储的，直接存储在_posts的目录。Hexo天生集成了markdown，我们可以直接使用markdown语法格式写博客，例如:hello-world.md。新增加一篇文章，就在_posts目录，新建一个xxx.md的文件。

themes目录：是存放皮肤的，包括一套Javascript+CSS样式和基于EJS的模板设置。通过在themes目录下，新建一个子目录，就可以创建一套新的皮肤，当然我们也可以直接在landscape上面修改。

全局配置Hexo

  1. 修改HexoBlog文件夹下的.config_yml全局配置文件

注意yml文件的严格格式：例 type:(这里必须有一个空格) git，其他亦如此。

  1. 每次修改完_config.yml的deploy都需要重新执行一次：npm install hexo-deployer-git --save

部署Hexo到github上

  1. 清除缓存：hexo clean （网页正常情况下可以忽略此条命令）

  2. 重新生成静态化页面：hexo g

  1. 注意以上命令都是在cmd下执行，但把项目部署到github上必须使用git bash，否则会报
    permission denied
    fatal：Could not read from remote repository
    Please make sure you have the correct access rights and the repository exists

  1. 鼠标选择HexoBlog文件夹，鼠标右键git bash here
    后输入：hexo d（或者hexo deploy）

至此Hexo已部署上了github，此时在浏览器输入http://dodomonster.github.io (注意把dodomonster换为你的github昵称)即可看到hexo

##为你的博客添加自定义域名##

每次进入都要通过输入http://yourname.github.io是不是很不爽？想不想跟别人一样有自己的个人域名，想那就继续看下去吧！

  1. 购买属于你自己的域名

推荐购买网站：

-阿里云
-godady

两者都支持支付宝付款。我选择的是阿里云，所以在此主要介绍阿里云购买以及解析。

  1. 打开阿里云，注册一个阿里云账号，登录后搜索到你想要的域名有没有被售出使用。成功购买到你想要的域名后，打开管理控制台点击域名，会查看到你刚刚买到的域名，点击管理

  1. 点击左边导航栏的解析设置后点击右边的进入高级设置

  1. 点击添加解析，添加框框内的那三条解析，注意把mrzzchao.github.io改为你对应的github pages地址

  1. 在自己的HexoBlog\source目录下创建一个CNAME文件(鼠标右键git bash here 输入命令：touch CNAME)，后使用文本编辑器，加入自己的域名地址，如：www.dodomonster.red,可以不加wwww。稍等10分钟左右打开浏览器输入你的域名就可以看到你自己的博客啦！！！

写博客

在一切都配置安装部署好，就可以愉快地写博客啦~~~

  1. 新建博客文件：新打开一个git bash命令行窗口，cd到/F/HexoBlog下，执行命令：hexo n “Blog破蛋记”，即会在以下目录生成Blog破蛋记.md文件


2.打开该文件使用sublime text或者atom或者notepad++但最好不要使用windows自带的记事本编辑，使用MarkDown语法来写博客即可

##写完文章后部署##

  1. 可以先在本地进行预览，在HexoBlog文件夹下命令行或者git bash输入以下命令：hexo s（或者hexo server），打开浏览器输入：http://localhost:4000

  2. 预览没有问题后，部署分两步：

    hexo g（或者hexo generate）
    hexo d（或者hexo deploy）

或者直接一步： hexo d -g
