AngularJS单元测试-Karma+Jasmine

Veröffentlicht am 2016-06-12

前言

入门AngularJS也算挺久了，但并没有运用到实际项目中。最近在看《AngularJS即学即用》这本书，感觉还不错，对入门挺好。但毕竟现在Angular2也出了蛮久了，但该书还是基于Angular1.2写的，尤其Angular1.2版本之后的变动和差异也蛮大的。不过已经既然已经开坑了还是想继续看完，也应该会有所收获。

什么是单元测试

众所周知，AngularJS的优点之一就是其单元测试简单得令人难以置信，也由于这样的特效大大提升了应用的稳定性及长期可维护性。

那么什么是单元测试呢？单元测试是指：将单个函数从代码中抽取出来，为它编写断言(assertion)以及测试用例来确保它的实际运行结果是符合预期的。

为什么在基于Javascript的客户端进行单元测试

  1. 正确性证明
  2. 弥补编译器的缺乏
  3. 尽早发现错误
  4. 防止回归错误
  5. 良好的说明方式

安装karma

karma(http://karma-runner.github.io/)能够让单元测试变得非常简单的同时其速度更是快得出乎预料。它通过NodeJS和SocketIO来进行测试。

  1. 安装NodeJS，安装配置可参考windows 10 下配置安装node.js

  2. 全局安装Karma CLI
    打开命令行输入命令：sudo npm install -g karma-cli

  3. 本地安装Karma，安装目录为需要运行的测试用例所在目录，命令：npm install karma

  4. Karmayeyou chaj ，本文主要介绍通过Jasmine和Chrome的加载插件，安装命令：npm install karma-jasmine karma-chrome-launcher

注意：3、4步骤的命令都必须运行在源代码所在的目录中

Karma配置

工欲善其事，必先利其器。在正式开始使用Karma之前，我们得定义好配置文件

module.exports=function(config){
    config.set({
        //放置文件的根目录
        basePath:'',

        //使用哪些测试框架(jasmine/mocha/qunit...)
        frameworks:['jasmine'],

        //浏览器需要加载的文件列表或者文件匹配表达式
        files:[
            'angular.min.js',
            'angular-mocks.js',
            'controller.js',
            'simpleSpec.js'
        ],

        //需要排除的文件列表或者文件匹配正则表达式
        exclude:[],

        //web服务器端口
        port:8081,

        //日志等级
        //合法值为：LOG_DISABLE||LOG_ERROR||
        //            LOG_WARN||LOG_INFO||LOG_DEBUG
        logLevel:config.LOG_INFO,

        //进行测试时是否允许随时监视文件编号
        autoWatch: true,

        //启动以下浏览器，目前可以兼容
        // - Chrome
        // - ChromeCanary
        // - Firefox
        // - Opera
        // - Safari(仅限Mac)
        // - PhantomJS
        // - IE(仅限Windows)
        browsers:['Chrome'],

        //持续集成模式
        //如果设置成true则会启动浏览器，运行测试然后退出
        singleRun:false

    });
};

参数解析：

  • basePath（根目录）
    basePath参数指定了所需要加载文件所处的目录，包括测试本身所需文件以及测试引用到的文件。该参数是一个相对路径，其根路径为Karma配置文件所处目录。

  • frameworks（框架）
    该参数指定了采用哪些框架，它是一个数字，可以加载多个框架。

  • files（文件）
    该参数指定了需要加载的文件名（或者文件路径），这也是一个数组。我们使用的是AngularJS，那么首先需要加载的是AngularJS的库，接着就是angular-mock.js，它是由AngularJS提供的一个辅助测试的库。最后，还需要加载应用的源文件以及实际的单元测试文件。

  • exclude（需要排除的文件）
    该参数指定了一组需要排除的文件。既可以使用通配符比如**.js，也可以指定完整的文件名比如karma.conf.js

  • port（端口）
    该参数指定了Karma测试运行在哪个端口上，默认值是8080（与tomcat的默认端口相同，请确认Tomcat没有占用该端口）

  • logLevel（日志等级）
    该参数指定了Karma需要收集浏览器哪一等级的日志（console.log，console.info）

  • autoWatch（自动监控）
    该参数是目前为止Karma最炫酷的功能，它告诉Karma监控所有files参数中包含的文件，如果它们中的任何一个更新了，就重新运行所有受影响的单元测试。如果该参数被设置为true，那么用户就不需要手动触发测试，Karma会自动进行监控。

  • browsers（浏览器）
    该参数指定了Karma初始化时打开哪个浏览器。大部分情况下，需要相应地加载插件（鉴于我安装的是chrome-launcher，就知道Chrome）

  • singleRun（是否重复运行）
    该参数是一个布尔值，决定了Karma在结束一次单元测试运行后是否会关闭服务器。在持续集成（continuous integration）环境下，该参数应该被设置为true，在其他环境可无视之。

Karma Configuration File Overview page

其实。。。上面描述的配置是可以自动生成的!!!一行命令就足以自动产生所需配置文件：karma init，输入命令后需要回答一些问题，按你自己的需要输入回答即可。

Blog破蛋记

Veröffentlicht am 2016-06-10   |   in settings

前言

经过1天的折腾终于把此博客搭建好并上线了，真心想说句不容易，躺了好多坑呀。。。因此也作为本博客的第一篇正式的博客记录下来。

前提物质条件

所谓“经济基础决定上层建筑”（好吧。。。纯属瞎扯）!!!本博客是通过Hexo和github pages搭建的。在使用Hexo搭建博客前首先你要在自己电脑上安装配置好以下三样东西。

  • NodeJS环境：安装配置可参考教程 windows下配置安装NodeJS

  • git：安装配置可参考教程 windows下配置安装git

  • github账号：请自行打开 github官网 进行注册登录

安装Hexo

当你的电脑上node环境配置安装好，git和github配置好后，方可正常又顺利地按照以下步骤对hexo进行配置安装。

  1. 打开命令行cmd，安装Hexo： npm install -g hexo
  1. 查询Hexo是否安装成功及版本：hexo -v

配置Hexo

  1. 创建本地文件夹：HexoBlog(名字随你起的。。。)，此处文件夹路径为：F:\HexoBlog。命令行下进入该文件夹后输入hexo init

  1. 安装依赖和插件：npm install

  2. 生成静态化页面：hexo generate（或者hexo g）

4.启动Hexo服务：hexo server（或者hexo s）

5.在浏览器输入：http://localhost:4000

至此简单的本地Hexo安装配置就成功了。

创建github仓库

  1. 打开https://github.com 登录github，新建仓库:yourusername.github.io(这是固定的仓库名，把yourusername换为你的github昵称，如我的github昵称为dodomonster，所以我的仓库名为dodomonster.github.io)

  1. 返回hexoBlog文件夹，查看文件结构内容如下：

  • scaffolds 脚手架，也就是一个工具模板
  • scripts 写文件的js，扩展hexo的功能
  • source 存放博客正文内容
  • source/_drafts 草稿箱
  • source/_posts 文件箱
  • themes 存放皮肤主题的目录
  • themes/landscape 默认的皮肤
  • _config.yml 全局的配置文件
  • db.json 静态常量

在这里，我们每次用到的就是_posts目录里的文件，而_config.yml文件和themes目录是第一次配置好就行了。

_posts目录：Hexo是一个静态博客框架，因此没有数据库。文章内容都是以文本文件方式进行存储的，直接存储在_posts的目录。Hexo天生集成了markdown，我们可以直接使用markdown语法格式写博客，例如:hello-world.md。新增加一篇文章，就在_posts目录，新建一个xxx.md的文件。

themes目录：是存放皮肤的，包括一套Javascript+CSS样式和基于EJS的模板设置。通过在themes目录下，新建一个子目录，就可以创建一套新的皮肤，当然我们也可以直接在landscape上面修改。

全局配置Hexo

  1. 修改HexoBlog文件夹下的.config_yml全局配置文件

注意yml文件的严格格式：例 type:(这里必须有一个空格) git，其他亦如此。

  1. 每次修改完_config.yml的deploy都需要重新执行一次：npm install hexo-deployer-git --save

部署Hexo到github上

  1. 清除缓存：hexo clean （网页正常情况下可以忽略此条命令）

  2. 重新生成静态化页面：hexo g

  1. 注意以上命令都是在cmd下执行，但把项目部署到github上必须使用git bash，否则会报
    permission denied
    fatal：Could not read from remote repository
    Please make sure you have the correct access rights and the repository exists

  1. 鼠标选择HexoBlog文件夹，鼠标右键git bash here
    后输入：hexo d（或者hexo deploy）

至此Hexo已部署上了github，此时在浏览器输入http://dodomonster.github.io (注意把dodomonster换为你的github昵称)即可看到hexo

##为你的博客添加自定义域名##

每次进入都要通过输入http://yourname.github.io是不是很不爽？想不想跟别人一样有自己的个人域名，想那就继续看下去吧！

  1. 购买属于你自己的域名

推荐购买网站：

-阿里云
-godady

两者都支持支付宝付款。我选择的是阿里云，所以在此主要介绍阿里云购买以及解析。

  1. 打开阿里云，注册一个阿里云账号，登录后搜索到你想要的域名有没有被售出使用。成功购买到你想要的域名后，打开管理控制台点击域名，会查看到你刚刚买到的域名，点击管理

  1. 点击左边导航栏的解析设置后点击右边的进入高级设置

  1. 点击添加解析，添加框框内的那三条解析，注意把mrzzchao.github.io改为你对应的github pages地址

  1. 在自己的HexoBlog\source目录下创建一个CNAME文件(鼠标右键git bash here 输入命令：touch CNAME)，后使用文本编辑器，加入自己的域名地址，如：www.dodomonster.red,可以不加wwww。稍等10分钟左右打开浏览器输入你的域名就可以看到你自己的博客啦！！！

写博客

在一切都配置安装部署好，就可以愉快地写博客啦~~~

  1. 新建博客文件：新打开一个git bash命令行窗口，cd到/F/HexoBlog下，执行命令：hexo n “Blog破蛋记”，即会在以下目录生成Blog破蛋记.md文件


2.打开该文件使用sublime text或者atom或者notepad++但最好不要使用windows自带的记事本编辑，使用MarkDown语法来写博客即可

##写完文章后部署##

  1. 可以先在本地进行预览，在HexoBlog文件夹下命令行或者git bash输入以下命令：hexo s（或者hexo server），打开浏览器输入：http://localhost:4000

  2. 预览没有问题后，部署分两步：

    hexo g（或者hexo generate）
    hexo d（或者hexo deploy）

或者直接一步： hexo d -g
